宿主页（模拟第三方网站）— chat-widget.js + data-api-key + cookie token

第三方接入只需一行 <script>：API Key 写在 data-api-key， chat-widget.js 在 iframe 内 client 启动时经 postMessage 把 key 注入（client 自身不内置 key）。 终端用户 token 从宿主页 cookie 读取（data-token-cookie，默认 mep-access-token）， 作为 X-User-Token 注入 client —— 对应后端 Key 配了 user_info_url 的 callback 验证模式。 
<script src="http://localhost:3001/chat-widget.js"
        data-chat-url="http://localhost:3001"
        data-api-key="af_live_xxx"
        data-token-cookie="mep-access-token"
        data-title="Agent Flow 助手"
        data-width="680px"></script>
可选：data-user-token（显式 token，优先级高于 cookie）、data-width/right/bottom/open-on-load。 JS API：AgentFlowChat.open() / .close() / .toggle()。
本地测试 cookie 注入：真实接入时 mep-access-token 由客户系统登录写入；本地无登录态，点下方按钮模拟。 写入后打开侧滑抽屉，握手会把该值作为 X-User-Token 透传给 client（可在 Network 面板核对请求头）。
