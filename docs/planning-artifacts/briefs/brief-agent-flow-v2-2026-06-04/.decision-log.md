# Decision Log — Agent Flow v2 Brief

## 2026-06-04

### D1: 产品方向从 MES 领域专注转向通用平台
- **背景**: 原产品简报定位为 MES 领域原生的 Agent 编排工具
- **决策**: 放弃 MES 专注，转向通用 AI Agent 编排平台
- **原因**: 架构设计（八大领域）本身就是通用平台的设计，不限于特定行业
- **影响**: 需要重新定义目标用户、差异化定位和竞品对比

### D2: 工作流是 Agent 的一种能力，不是独立系统
- **背景**: 传统平台中 Agent 和 Workflow 是两个独立概念
- **决策**: 工作流作为 Agent 的可配置能力之一，Agent 可以自主决定调用
- **原因**: 统一 Agent 的执行模型——自主推理和确定性流程是同一Agent 的不同执行模式
- **影响**: 架构设计需要体现"Agent 自主选择工作流"的机制

### D3: 双角色用户模型——开发者建设，业务系统消费
- **背景**: 需要明确谁在使用平台
- **决策**: 开发人员在 Web 前端管理 Agent/工作流，业务系统通过 API/SDK 接入
- **原因**: 平台定位为 Agent 能力中台，不是终端用户产品
- **影响**: 前端面向开发者设计，API/SDK 面向系统集成设计

### D4: 部署模式——私有化内部使用
- **决策**: 暂时私有化部署，整个公司多部门使用
- **原因**: 内部工具定位，不需要 SaaS 多租户
- **影响**: 不需要考虑计费、公网暴露、多租户隔离
