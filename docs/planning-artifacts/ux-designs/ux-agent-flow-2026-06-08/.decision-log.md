## Discovery Decisions

### D1. 目标用户群
**A — 双角色：平台开发者（30+，技术背景强）+ 业务操作员（25-40，技术背景中等）**
- 开发者：构建/调试 Agent，需要技术细节（节点配置、模型参数、工具注册）
- 操作员：消费 Agent 能力，需要直观对话 + 简洁工作流触发

### D2. 设备形态
**A — Web 为主**（桌面 90% / 平板 10% / 手机几乎无）
- 决策原因：工业用户多在工位/控制台用大屏，手机只做轻量查看

### D3. 视觉风格方向
**A — 专业 / 工程 / 克制**
- 参考对象：Linear、Notion、LangChain LangSmith
- 关键特征：清晰层级、低饱和、强功能、信息密度高、留白克制
- 不取方向：Dify/Flowise 的活泼卡通风（会显得不够"工业"）

### D4. 品牌色
**A — 深蓝 + 青色辅色**
- 主色（Primary）：`#1E5EFF`（深蓝，专业、信任）
- 辅色（Accent）：`#00D4FF`（青色，AI 感）
- 警示色：标准 AntD 红/橙/绿
- 决策原因：AI Agent 编排 = 工程 + 智能；深蓝传达"工程"，青色传达"AI"

### D5. 主题模式
**A — 亮色主，暗色延后**
- MVP 阶段只做亮色主题
- 预留 token 化（CSS variables）便于后续扩展暗色
- 决策原因：MVP 50 用户内网部署，亮色对工位环境足够

### D6. 信息密度
**A — 中密度**
- AntD 默认（compact: false）
- 表格行高 40px（AntD middle）
- 段落间距 16px（AntD lg）
- 决策原因：内部系统高密度会"挤"，低密度会"慢"，中密度是工业 B 端甜点

### D7. 关键界面优先级
**A — 4 个核心界面（MVP 必做）**
1. **Agent 列表 + 详情**（`features/agent_management/`）
2. **Agent 配置表单**（能力组合：工具/知识库/工作流）
3. **工作流编辑器**（@xyflow/react 拖拽式 DAG）
4. **对话界面**（WebSocket 流式，含工作流触发）

延后界面（Post-MVP）：执行日志、工具注册中心、知识库管理、用户权限、API Key 管理、模型路由配置

### D8. 微交互/动效
**A — 克制动效**
- 过渡时长：200-300ms
- 缓动：cubic-bezier(0.4, 0.0, 0.2, 1)（Material standard easing）
- 范围：按钮 hover/active、Modal/抽屉 进出、Tab 切换、loading spinner
- 不做：页面级切换动画、列表项入场动画、复杂 micro-interaction
- 决策原因：工业 B 端用户要"快"不要"炫"

### D9. 无障碍（A11y）
- 基础合规：WCAG 2.1 AA 级
- 键盘导航：全功能可仅用 Tab/Enter/Esc
- 屏幕阅读器：AntD 默认有 ARIA 属性，工作流编辑器需补键盘可达性
- 色彩对比：所有文字 vs 背景 ≥ 4.5:1

### D10. 国际化（i18n）
- MVP 阶段：仅中文（与 `document_output_language` 对齐）
- Token 预留：所有可见文字走 i18n key，便于未来加英文
- 决策原因：内部工业系统多语言需求后置

### D11. 响应式
- 断点（与 AntD 一致）：
  - `xs` < 576px — 不支持（MVP 桌面优先）
  - `sm` ≥ 576px — 极简对话界面可用
  - `md` ≥ 768px — 平板可用，简化导航
  - `lg` ≥ 992px — **主力断点**（大多数工位屏）
  - `xl` ≥ 1200px — 工作流编辑器舒适区
  - `xxl` ≥ 1600px — 大屏，3 列布局可选

### D12. 浏览器兼容
- MVP：Chrome / Edge 最新两个版本（覆盖工业 PC 95%+）
- 不支持：IE、Safari（移动端）
- 决策原因：内网环境统一 Chromium 系
